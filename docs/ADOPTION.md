# Adopting this framework in your project

This repository is a **framework** for contract-first, partition-bound autonomous improvement. Your product code lives in **your** layout; this repo supplies rules, scripts, docs, and a **default root scaffold** (`api_spec/`, `server/`, `client/`) you can replace or repoint with the wizard.

## Operational model (theory → practice)

0. **Existing project?** Run **`python3 scripts/install_framework.py --target /path/to/your/repo`** once (from a clone of this framework). See **`docs/INSTALL_IN_YOUR_REPO.md`**.
1. Run **`python3 scripts/configure_synthesis.py`** — sets paths, goals, model pre-flight questions, CI skips, and syncs `partitions.json`.
2. Read **`docs/CONSOLIDATED_MODEL.md`** — single reconciled spec (authority, routing, loops, anti-corruption, fitness, evolutionary mechanics).
3. In every agent session, include **`SYNTHESIS_PROJECT.md`** (generated by the wizard).
4. Train agents with **`docs/PROMPTS.md`** (paste per session) and **`docs/AGENT_PLAYBOOK.md`** (procedure); use **`docs/SCENARIOS.md`** for edge cases.
5. Keep **Cursor** rules in `.cursor/rules/` aligned with your paths (globs); keep **`CLAUDE.md`** in the repo root for Claude Code.

## Already using Cursor or Claude Code?

Run the wizard with **scan** enabled so **`prior_agents`** is populated and **`SYNTHESIS_PROJECT.md`** lists existing rules/memory paths. See **`docs/MIGRATION_FROM_PRIOR_AGENTS.md`**.

## What you copy or merge

1. **`.cursor/rules/`** — Cursor agent authority and partition scopes (adjust globs to your paths).
2. **`CLAUDE.md`** — Claude Code entry (optional; customize prompts).
3. **`docs/`** — Architecture and workflows (trim or extend for your team).
4. **`scripts/`** — Partition check, OpenAPI validation, completeness scan, integration smoke.
5. **`synthesis/partitions.json`** — **You must edit this** to match your repository.

## Configure `synthesis/partitions.json`

### 1. Partitions

Set `partitions.backend.paths`, `partitions.frontend.paths`, and `partitions.contract.paths` to directories you own (examples: `services/api/`, `apps/web/`, `contracts/openapi/`).

### 2. Reference example (CI targets)

`reference_example` tells CI scripts **where** to run validation, tests, and smoke **in this framework repo**. In a consumer monorepo, point these at your real service and app:

```json
"reference_example": {
  "openapi_spec": "contracts/openapi.yaml",
  "backend_dir": "services/api",
  "frontend_dir": "apps/web",
  "integration_app": "myapp.main:app",
  "integration_port": 8765
}
```

- **`integration_app`**: Uvicorn import string for your ASGI app (if you use something else, replace `scripts/integration_smoke.py` with your runner).
- **`integration_port`**: Free port for CI smoke.

### 3. Framework and shared paths

- **`framework_paths`**: Directories that are **not** product partitions (this framework’s `.cursor/`, `docs/`, `scripts/`, etc.). Add your own infra paths if agents may touch them.
- **`shared_readonly_paths`**: Same idea for read-only or policy paths; **include your root `README.md`, `.gitignore`, and CI** if agents should not treat them as violations.

### 4. Reference example exemption (optional)

`reference_example_prefixes` lists subtrees where **demo or sample** code may touch backend + contract + frontend in one commit without tripping the per-commit partition rule. **Remove or empty** this in a strict consumer repo, or set it to your `examples/` tree only.

### 5. Legacy path allowlist (optional)

If you **moved** partitions (for example from repo root into `examples/`), old paths may still appear in `git diff` against `main`. Add those prefixes to `legacy_layout_prefixes` so the unknown-path check stays quiet until the old history is no longer in range. Remove the entries once rebased or after the old layout is gone.

## Wire CI

Copy `.github/workflows/synthesis-ci.yml` and set:

- **`working-directory`** for backend steps to your `reference_example.backend_dir`.
- **`working-directory`** for client steps to your `reference_example.frontend_dir`.
- **`cache-dependency-path`** to your frontend lockfile path.

## Operational loop (reminder)

1. **Contract** change → semver bump → codegen.
2. **Backend** commit/PR implements the spec.
3. **Frontend** commit/PR consumes generated SDK.
4. **Integration** verifies real HTTP and scenarios.

Never weaken validation to “make green”; route failures per `docs/WORKFLOWS.md`.
